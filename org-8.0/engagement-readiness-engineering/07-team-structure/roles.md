# ERE Roles

[← Back to Team Structure](README.md) · [← Back to ERE Guide](../README.md)

---

This document describes the roles that comprise the ERE team.

---

## Core Team Roles

| Role | Responsibility |
|------|----------------|
| **Product Manager** | Roadmap, prioritization, stakeholder alignment; owns OKRs and success metrics |
| **Engineering Lead** | Technical direction, architecture, delivery; code quality and system reliability |
| **Engineers** | Build and maintain systems; implement features, fix bugs, improve performance |
| **Designer** | UX for internal tools and customer portal; ensures consistent, usable interfaces |
| **Knowledge Engineer** | Knowledge base system, taxonomy, quality standards, curation workflows |
| **AI/ML Engineer** | Agent development, fine-tuning, performance monitoring; agent governance implementation |

---

## Detailed Role Descriptions

### Product Manager

**Focus:** What to build and why

| Area | Activities |
|------|------------|
| Roadmap | Define phases, prioritize features, sequence investments |
| Stakeholder alignment | Gather requirements from EPMs, EAs, Sales, ERC; communicate priorities |
| OKRs | Define and track success metrics; report progress to ERC |
| User research | Understand tool adoption patterns, pain points, opportunities |

**Interfaces with:** ERC leadership, EPMs, EAs, Engineering Lead

### Engineering Lead

**Focus:** How to build it well

| Area | Activities |
|------|------------|
| Architecture | System design, integration patterns, technology selection |
| Delivery | Sprint planning, technical decision-making, release management |
| Quality | Code review standards, testing strategy, reliability targets |
| Team development | Mentoring engineers, technical skill growth |

**Interfaces with:** Product Manager, Engineers, AI/ML Engineer, platform teams

### Engineers

**Focus:** Building and maintaining systems

| Area | Activities |
|------|------------|
| Development | Feature implementation, bug fixes, performance optimization |
| Operations | Monitoring, incident response, system maintenance |
| Documentation | Technical documentation, API specs, integration guides |
| Inner source | Reviewing contributions from rotating engineers |

**Interfaces with:** Engineering Lead, Designer, Knowledge Engineer

### Designer

**Focus:** User experience across all ERE tools

| Area | Activities |
|------|------------|
| Internal tools | UX for Proposal Kit, BRD Author, Governance Prep, etc. |
| Customer portal | Portal design, Concierge interaction patterns |
| Design system | Consistent components, patterns, and guidelines |
| User research | Usability testing, feedback synthesis |

**Interfaces with:** Product Manager, Engineers, EPMs (as users)

### Knowledge Engineer

**Focus:** Knowledge as a first-class function

| Area | Activities |
|------|------------|
| System ownership | Knowledge base infrastructure, taxonomy, search |
| Quality standards | Completeness criteria, reusability scoring, findability (tagging) |
| Curation workflows | Review processes, consolidation, gap identification |
| Steward coordination | Onboarding and guiding rotating Domain Stewards |

**Interfaces with:** Domain Stewards, EPMs, Pattern Curator Agent

### AI/ML Engineer

**Focus:** Agents that assist and automate

| Area | Activities |
|------|------------|
| Agent development | Building Proposal Agent, BRD Agent, Concierge, etc. |
| Fine-tuning | Training agents on organizational data and patterns |
| Performance monitoring | Accuracy tracking, escalation rate analysis |
| Governance implementation | Autonomy levels, audit trails, feedback loops |

**Interfaces with:** Engineering Lead, Knowledge Engineer, ERC (for autonomy reviews)

---

## Rotating Roles

### Domain Stewards

**Definition:** SMEs from Engagements who validate and enrich domain-specific content on a rotating basis.

| Area | Activities |
|------|------------|
| Content validation | Review and approve domain-specific knowledge artifacts |
| Enrichment | Add context, correct errors, identify gaps |
| Pattern identification | Flag reusable patterns from Engagement experience |
| Taxonomy input | Suggest taxonomy updates based on domain evolution |

**Rotation:** Typically 1-2 quarters; drawn from experienced EA, EPO, or EL roles.

**Reports to:** Knowledge Engineer (functionally)

## Role Count by Maturity

| Role | Initial State | Mature State |
|------|---------------|--------------|
| Product Manager | 1 | 1 |
| Engineering Lead | 1 | 1 |
| Engineers | 3-5 | 2-3 (core) + contributors |
| Designer | 1 | 1 (possibly shared) |
| Knowledge Engineer | 1 | 1-2 |
| AI/ML Engineer | 1 | 1-2 |
| Domain Stewards | 0 | 2-4 (rotating) |
