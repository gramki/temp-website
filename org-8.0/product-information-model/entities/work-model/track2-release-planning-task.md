# Release Planning Task

**Model:** Work Model
**Track:** Track 2: The Build Track (Construction)
**Owner:** Product Management, Tech Lead

## Definition

Work to scope a Customer Release — determining which PSDs/Initiatives are included, defining timeline and milestones, and allocating team capacity.

## Purpose

Makes the release scoping work explicit in the Build Track. A Customer Release (Dimension 1) requires deliberate planning to determine what is included, when it will be delivered, and how teams are allocated. This is the bridge between strategic intent (Initiatives) and tactical execution (Epics/Stories).
