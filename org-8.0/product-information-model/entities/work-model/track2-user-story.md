# User Story

**Model:** Work Model
**Track:** Track 2: The Build Track (Construction)
**Owner:** Developers, QA

## Definition

An Agile requirement delivering specific value — written from the perspective of a user persona, describing a discrete unit of functionality.

## Purpose

Translates Epics into user-facing increments of value. User Stories are the primary planning unit for sprint-level work.
