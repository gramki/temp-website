# Module / Domain

**Model:** Definition Model
**Dimension:** Dimension 8: The Structural Dimension (Topology)
**Owner:** Enterprise Architects, Business Analysts

## Definition

A major bounded context within the product — a logically cohesive area of functionality.

## Purpose

Provides the primary decomposition boundary below the Product level. Modules contain Capabilities and align with Subsystems (Dimension 5) and Data Domains (Dimension 9).
