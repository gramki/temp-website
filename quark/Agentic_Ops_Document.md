
# Agentic Operations — Comprehensive Document

Below is a comprehensive, expanded catalog of bank back-office operations functions, grouped into clear domains. Your original list is fully included and extended to cover what most retail, business, commercial, and treasury banks classify as operations.

This is suitable for creating an Agentic Operations Map later if you want one.

## Expanded Bank Back-Office Operations Functions

## 1. Agentic Automation Heat-Map

### Overview
Agentic automation potential varies across banking back-office operations using the following scale:

★★★★★ – Very High  
★★★★☆ – High  
★★★☆☆ – Medium  
★★☆☆☆ – Low  
★☆☆☆☆ – Very Low  


(15 functions × 100+ individual operations)

⸻

### 1. Customer Lifecycle & Identity Operations

| Operation                                       | Rating   | Reason                                                                                  | Agent Types                                 |
|-------------------------------------------------|----------|-----------------------------------------------------------------------------------------|----------------------------------------------|
| KYC / Customer Due Diligence (CDD)              | ★★★★☆    | High structure, rules-driven, document-heavy. Agents excel in extraction + rule checks.  | Do-Agents (extraction), Guardrail-Agents (CDD rules) |
| Enhanced Due Diligence (EDD)                    | ★★★☆☆    | Narrative-building, judgement, long-form risk assessment needs human review.             | Think-Agents                                 |
| Customer Onboarding Ops                         | ★★★★★    | Fully automatable: document checks → rule validation → account creation.                 | Do-Agents                                    |
| Periodic KYC Refresh                            | ★★★★★    | Repeatable; record comparisons + alerts + updates + communication cycles.                | Orchestrator + Do-Agents                     |
| Identity Verification (Video KYC / Biometric)   | ★★★★★    | AI perception + liveness detection increasingly commoditized.                            | Do-Agents with ML sub-systems                |
| Customer Record Maintenance                     | ★★★★★    | Demographic updates, nominee updates, FATCA/CRS updates are deterministic.               | Do-Agents                                    |

## 2. Agentic Ops Fabric Architecture

### Layer 1: Operational Data Plane
Core systems of record:  
- Core Banking  
- Cards  
- Payments  
- Trade Finance  
- Treasury  
- GL/Finance  
- CRM/Case Systems  
- Document repositories  

### Layer 2: Intelligence & Automation Layer
- LLM Reasoning  
- ML Scoring Models  
- Rules Engine  
- OCR & Data Extraction  
- Vector Stores  
- Anomaly Detection  

### Layer 3: Agentic Operations Fabric
**Do-Agents** (execution):  
- Reconciliation bots  
- Settlement bots  
- KYC update bots  
- Loan-booking bots  

**Think-Agents** (reasoning):  
- Fraud investigation  
- AML triage  
- Underwriting support  

**Orchestrator Agents**:  
- Workflow management  
- SLA tracking  

**Guardrail Agents**:  
- Compliance checks  
- Policy enforcement  

### Layer 4: Human Ops Interface
- Workbench  
- Dashboards  
- Approvals  
- Supervisory controls  
- Audit logs

## 3. Role-by-Role Decomposition of Each Operational Stream

### Reconciliation Analyst
Agents can:  
- Ingest statements  
- Classify breaks  
- Autoresolve known patterns  
- Escalate exceptions  

**Coverage:** 85–95%

### Dispute/Chargeback Officer
Agents can:  
- Interpret scheme rules  
- Prepare evidence  
- Draft dispute documents  

**Coverage:** 70–90%

### KYC Ops Analyst
Agents:  
- Extract data  
- Validate rules  
- Risk score  
- Update systems  

**Coverage:** 90–100%

### AML/Fraud Investigator
Agents:  
- Gather evidence  
- Perform link analysis  
- Draft case narratives  

**Coverage:** 65–85%

### Loan Operations Officer
Agents:  
- Validate documents  
- Check collateral  
- Set up loan accounts  
- Generate repayment schedules  

**Coverage:** 70–90%
