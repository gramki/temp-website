# Quick Start Guide — Roles & First‑Week Playbook

Purpose: Give each persona a practical, skimmable start — what to open daily, which forums to run/attend, primary signals to watch, and first‑week actions. Use with Section 9 (Operational Governance), Appendix A/B (dashboards/alerts), Appendix E (rituals), Appendix H (audit), and Appendix M (templates).

Conventions
- Open daily: dashboards by audience in §9.13 and Appendix A
- Escalate with options: use Appendix M (decision/exception templates)
- Chair/ownership: label EO vs Studio in invites (§9.1)

---

## Engagement Org

### Engagement Owner (EO)
- Daily: Steering queue (decision backlog age), FVS/Capacity bands, top 5 risks/options
- Weekly/Monthly: Review Monthly pack; pre‑wire Steering with 2–3 options
- Primary signals: FVS/Capacity bands; unfunded exposure; policy/threshold breach trends
- First week
  - [ ] Confirm forum ownership/chairs (EO vs Studio) and decision SLA (≤3 business days)
  - [ ] Align on model (SFM/TCM) and contract appendix checkpoints (Appendix J)

### Commercial Manager
- Daily: Unfunded exposure; forecast vs actual (last 2 cycles); dependency financial risk
- Weekly: CCB/Commercial Review; CR lifecycle hygiene; funding ledger updates
- Primary signals: Unfunded exposure ≤ threshold; delivery vs dependency variance split
- First week
  - [ ] Wire Commercial dashboard; verify DependencyFunding% register exists
  - [ ] Publish CR decision rights and SLA to program

### Account Manager
- Daily: Escalations from Commercial/Steering; SoW/appendix changes in flight
- Weekly: Customer comms on funded/unfunded scope; standby protections (TCM)
- Primary signals: Open commercial risks; decision backlog age; idle‑time protections
- First week
  - [ ] Align escalation ladders and comms templates (Communication_Templates appendix)

## Studio Org

### Studio Owner
- Daily: Program health summary; breach roll‑up across requirements/quality/commercial
- Weekly/Monthly: Sponsor alignment; unblock cross‑team constraints; policy tuning
- Primary signals: Repeated breaches; accumulation > amortization; decision latency
- First week
  - [ ] Confirm reserves (quality/debt 15–25%); publish “operating defaults” (Appendix T)

### Delivery Manager (DM)
- Daily: Team/Program dashboards (WIP/aging, CI/test state, dependency calls)
- Weekly: Run Weekly Health & Risk Review; Debt Clearance Review; prep Monthly pack
- Primary signals: %RfP & shelf‑life; pass rate/error budget/flake; FVS/Capacity; dependency variance; amortization ≥ accumulation; decision backlog age
- First week
  - [ ] Complete Flight Check (ledgers, dashboards, RfP gate)
  - [ ] Publish ritual calendar with chairs; create decision log; post escalation ladders

### Delivery Product Owner (DPO)
- Daily: RfP throughput and shelf‑life; integration readiness index; AC/NFR hygiene
- Weekly: Co‑chair RfP Readiness Workshop; design/impact addenda sign‑off
- Primary signals: %RfP ≥ target; features scoped to subsystem; contracts/creds/env ready
- First week
  - [ ] Seed MAR/RfP templates; align AC/NFR examples; index Risk Surfaces

### Delivery Product Manager (DPM)
- Daily: Ready coverage; spike outcomes; traceability health
- Weekly: Backlog Grooming/Refinement; support RfP readiness
- Primary signals: DoR met; Ready coverage ≥ target; spikes closed on time
- First week
  - [ ] Wire decomposition labels/queries; adopt AC templates; Ready checklist visible

### Engineering Manager (EM)
- Daily: CI/test state; flake; critical‑path pass rate; WIP/aging
- Weekly: Capacity bands vs commitment; hardening windows when error budget red
- Primary signals: Pass rate ≥ target; flake ≤ threshold; commitment within bands
- First week
  - [ ] Enforce reserve; add pre‑merge component test gates on critical paths

### Tech Lead / Principal Engineer
- Daily: Contract tests and component suites; integration stubs/contracts
- Weekly: Risk‑driven test additions; debt items linked to signals
- Primary signals: Contract drift; idempotency/ordering; latency/chaos coverage
- First week
  - [ ] Validate integration readiness checklist (5.10); publish runbooks

### QA/Test Lead
- Daily: Critical‑path pass rate; leakage trend; flake; pipeline stability
- Weekly: Quality Council; stabilization plan and quarantine window
- Primary signals: Error budget; pass rate; flake; change failure rate
- First week
  - [ ] Wire Allure/Jira traceability; set gates and budgets; publish test matrix

## Customer Org (for alignment)

### Customer PO/PM
- Daily: RfP queue; acceptance traceability; integration readiness statuses
- Weekly: Attend RfP workshop; confirm AC/NFR; sign‑off or actions
- Primary signals: %RfP, shelf‑life, AC completeness; dependency readiness evidence
- First week
  - [ ] Agree on MAR/RfP templates; confirm decision SLA; align on dashboards to consume
