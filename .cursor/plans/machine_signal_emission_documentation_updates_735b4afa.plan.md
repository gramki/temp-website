# Machine Signal Emission Documentation Updates

> **Status:** ✅ **COMPLETED** - All tasks implemented and verified

> **Completion Date:** 2026-01-15

> **Verification:** Cross-references checked, inconsistencies resolved

## Overview

This plan updates Hub documentation to incorporate resolved design decisions about how Machines produce signals through Signal Providers. The changes span Machine Registry, Signal Providers, schemas, and integration guides.

## Architecture Decisions Summary

Key resolved decisions:

- Multi-provider signals are not deduplicated by Hub
- Provider selection is Machine's choice (outside Hub scope)
- Schema validation occurs at Signal Provider
- Hub ingress endpoints are subscription-scoped, per-workbench, provisioned by tenant admin/authorized developers
- Hub-hosted topics use pattern: `/hub/{tenant}/{subscription}/{workbench-id}/{signal-provider}/{name-slug}`
- Pull-to-push conversion is automatic when endpoints are provisioned
- SFTP pull doesn't validate files, just reads and pushes

## Implementation Notes

### Schema Validation

- All signal schema validation occurs at Signal Provider during normalization
- Each provider validates according to its protocol (OpenAPI for Webhook, CloudEvents for Atropos, File Format Spec for SFTP)

### Endpoint Provisioning

- Hub ingress endpoints are subscription-scoped and per-workbench
- Pattern: `/hub/{tenant}/{subscription}/{workbench-id}/{signal-provider}/{name-slug}`
- Provisioned by tenant admin or authorized developers as resources

### Pull-to-Push Conversion

- Automatic when endpoints are appropriately provisioned and specified
- Hub-hosted topics are dedicated to machine instance
- Auto-provisioned with machine instance, managed by tenant admin
- Lifecycle tied to machine instance

### Multi-Provider Support

- Machines can emit through multiple providers simultaneously
- Hub does not deduplicate or acknowledge as same/redundant signals
- Each signal is processed independently

### Provider Selection

- Machine's choice, outside Hub scope
- Machines are represented in Hub, not defined in Hub (often external systems)
