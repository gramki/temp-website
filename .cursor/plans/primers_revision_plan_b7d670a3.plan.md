```
# CIO Primer: Hub for Information-Centric Work

## Executive Summary
- Most enterprise work is information-centric
- AI adoption requires a new model for how work operates
- Hub provides operational infrastructure for governed AI-Human collaboration

## What Is Information-Centric Work?
- Definition: Work where inputs, transformations, and outputs are information
- Examples: Customer service, finance, HR, IT, compliance, knowledge work
- The insight: This is most of your enterprise's work
- NOT: Manufacturing floors, physical logistics (though planning/coordination is)

## Why Information-Centric Work Requires a New Model

### The Traditional View (and Its Limits)
- Work modeled as: tasks, records, systems, procedures
- Systems built around: data storage, workflow steps, user roles
- AI integration attempt: add AI as another tool in the chain
- Result: AI remains isolated, underutilized, unaccountable

### The Paradigm Shift
- Work is not tasks — it's situations that need resolution
- Situations require judgment, not just execution
- Multiple agents (human, AI, rules) must collaborate toward goals
- The organization must learn from outcomes

### Why This Matters for AI Adoption
| Traditional Model | Information-Centric Model |
| AI as a tool to call | AI as an agent that collaborates |
| Workflow assigns tasks | Scenario defines goals |
| Human supervises AI | Human and AI work together |
| Audit logs actions | Audit captures reasoning |
| Each system has its own AI | Unified AI-Human operational model |

### The Insight
- Enterprises already do information-centric work — they just model it wrong
- The right model: signals → situations → collaborative resolution → learning
- This model naturally accommodates AI because it's inherently multi-agent

## What Hub Provides

### An Operational Platform for Collaborative Problem-Solving
- Workbenches: Domain-specific operational environments
- Scenarios: Situations that need resolution (goal-oriented, not procedure-oriented)
- Requests: Collaboration surfaces where agents work together
- Agents: Humans, AI, rules, workflows — unified participation model

### Practical Use Cases
- Operations centers (payments, claims, service)
- Case management (disputes, compliance, HR)
- Knowledge work coordination (analysis, review, decisions)
- AI-augmented processes (any scenario where AI can contribute)

### The Infrastructure That Makes It Work (Four Pillars)
- Context that grounds agents in your domain
- Structure that guides when/how agents engage
- Memory that accumulates organizational learning
- Governance that ensures trust and accountability

## Hub + Seer: The Two-System Architecture
- Hub governs operations (scenarios, requests, collaboration)
- Seer governs AI agents (identity, runtime, capabilities)
- Together: Trusted AI-Human collaboration at scale

## Enterprise Capabilities
- Multi-tenancy, security, compliance
- Cognitive Audit Fabric for AI accountability
- Agent lifecycle management

## What Hub Is (and Isn't)
- Hub IS: Operational platform for governing collaborative work
- Hub IS NOT: Replacement for ERP/CRM/core systems

## Getting Started
- Pilot approach: one domain, 2-3 scenarios, measure outcomes

## Next Steps
- Links to foundational-beliefs.md, vision.md, applicability-guide.md
```

# Developer Primer: Build What Matters

## The Frustration You Know
- You build the same plumbing over and over (auth, routing, notifications, audit)
- Infrastructure takes more time than business logic
- AI integration is ad-hoc — different for every project
- "Human-AI handoff" means emails and hope
- Your code runs, but you can't see its operational impact
- Testing operational scenarios requires standing up the world

## The Shift: Platform Handles Infrastructure, You Handle Intelligence

### What You Stop Building
- Signal routing and normalization
- Task queues and assignment logic
- Notification systems
- Audit infrastructure
- Identity and access control
- Data stores per application

### What You Focus On
- Business logic for your Scenario
- Decision-making that matters
- Integration with external systems (via Tools)
- AI agent behavior (via Seer)

## What This Means for You

### Your Code Has Context
- You receive Requests with full operational context
- You know the Scenario, the goals, the history
- You're not building in isolation

### Your Work Connects to Something Larger
- Process Architect designs the "what" — you implement the "how"
- Supervisor configures the deployment — your code runs in their context
- Your Hub Application is one part of governed collaboration

### AI Is a First-Class Concern
- AI agents are participants, not afterthoughts
- Employment specs define autonomy boundaries
- You build for human-AI collaboration, not around it

### You Can See the Impact
- Requests flow through your application
- Tasks get assigned, completed, escalated
- Outcomes are visible, auditable

## What You Can Now Do

| Before | Now |
| Build routing, auth, audit yourself | Platform provides infrastructure |
| Integrate AI ad-hoc | AI agents are first-class |
| Hope your code connects to process | Your code implements Scenarios |
| Test in isolation | Test against operational flows |
| Deploy and pray | GitOps promotion with visibility |
