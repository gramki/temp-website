
Case Management in Agentic World

A Case Automation System
- Trigger invokes the automation system through the Signal, Scenario, and Environment parameters along with any specific parameters specified in the trigger defintion itself
- The Automation systems setup a Case and provides the Case Entity to Hub to present this information wherever appropriate
- The Automation sytems manages the life of the Case Entity until it is closed

# Seer Case Automation System

## Conversation Case Model
- Integrates Case entity into MS Teams as a Chat Group
- Provides a Case Webview that can be embedded in various surfaces to enable participants in resolving a case
- Integrates Case entity with Jira Task provides updates there

- Case Specification 
-- Goals for the case
-- Agents to be involved -- directly or through a notion of a task-queue
-- Translation of Case Updates to Signals for the Agents (Agents may be AI Agents, Procedure following automation applications, or Human Agents)
-- Knowledge Base, SOPs to be used for the Case
-- Extensions to the Case View
  
Case Coordinator
Collaboration Channels - MS Teams, Email, Custom Application Interfaces
Role Playing Agents
Case Context (Session Memory) - Chat Log, Key/Value Store with various scopes - case, agent

## Jeeves Cookbook Case Model
- Agents and humans can add cells to jupyter notebook to solve problem
- They can start from a specific base notebook and add additional cells as they proceed
- Humans can access these notebooks from their IDEs while resolving a problem
