# Research: US Security Standards

**Research ID:** R4.4  
**Date:** 2024-12-19  
**Purpose:** agents-and-their-roles.md - Architecture

## Research Question

What are PCI-DSS, SOC 2, and other security requirements for US banking?

## Key Findings

US banking must comply with PCI-DSS (card data), SOC 2 (security controls), and other standards. Multi-factor authentication, encryption, and audit trails are required.

## Data Points

| Standard | Scope | Key Requirements |
|---------|-------|------------------|
| PCI-DSS | Card data | Encryption, access controls |
| SOC 2 | Security controls | Security, availability, processing integrity |
| FFIEC | Banking | Multi-factor authentication, encryption |
| NIST | Cybersecurity | Framework for security |

## Detailed Findings

### PCI-DSS

**Payment Card Industry Data Security Standard:**
- Required for card data handling
- Encryption requirements
- Access controls
- Regular audits

**Agent Implications:**
- Card data must be encrypted
- Secure storage required
- Limited access
- Audit trails

### SOC 2

**Security Controls:**
- Security
- Availability
- Processing integrity
- Confidentiality
- Privacy

**Requirements:**
- Regular audits
- Control documentation
- Incident response
- Continuous monitoring

### FFIEC Guidance

**Multi-Factor Authentication:**
- Required for online banking
- Multiple factors
- Strong authentication

**Encryption:**
- Data in transit
- Data at rest
- Strong encryption standards

The agent architecture must meet all security standards. This includes encryption, authentication, access controls, and audit trails. Security is built into the architecture.
