# Reading List

> **Status: empty.** This is a curated list, not a pre-populated one. Entries are added as the team reads the relevant material first-hand. Do not seed this file from memory or AI summarization — that would defeat its purpose.

## How to add an entry

When you read a paper, talk, post, book chapter, or repository that is relevant to ACE's posture (see [se-3-0.md](se-3-0.md)), add a row below with:

| Title | Author(s) | Year | Theme | Format | Relevance to ACE |
|---|---|---|---|---|---|
|   |   |   |   |   |   |

**Theme** should be one of: agentic SDLC, multi-agent orchestration, AI-augmented requirements engineering, prompt programming, AI-generated code governance, human–agent teams, broader SE 3.0 positioning. Add new themes as needed.

**Format** should describe medium: paper, talk, book chapter, blog, repo, technical report.

**Relevance to ACE** is a sentence or two — not a summary of the source. Say specifically what about ACE this source confirms, contradicts, or extends.

## Conventions

- Cite primary sources when possible. If you are quoting a paper via someone else's blog post, cite the paper.
- Include a stable URL or DOI.
- If the source is paywalled and we have institutional access, note that.
- If you are unsure whether a source belongs here, add it to [se-3-0.md](se-3-0.md) "to triage" first; promote to this list only after reading.
