# Operating Model

The Operating Model describes **how the organization executes**. It covers the coordination patterns and organizational design that make the Work Model usable in practice.

This folder starts the Operating Model with governance because governance is where coordination, authority, evidence, policy, rituals, dashboards, and continuous improvement meet.

## What the Operating Model contains

- Ritual definitions and cadences
- Governance policies and evidence requirements
- Participant roles and decision authority
- Operating guidance and playbooks
- Reports and dashboards that support rituals
- Register definitions for risk, debt, compliance, and recognition
- Patterns for evolving the operating model itself

## What it does not contain

- Product structure and product strategy entities — those live in the Definition Model.
- Work entities and orchestration items — those live in the Work Model.
- Runtime execution details — those belong to the Foundry Platform implementation.

## Relationship to the other models

```text
Definition Model
  defines what the product is and intends to become

Work Model
  defines what work exists and how work items move

Operating Model
  defines how the organization executes that work
```

The Operating Model depends on the Work Model: you need to know what work exists before you can define rituals, roles, cadences, policies, and evidence expectations around it.
