# Content Style (v0.1)

Defines writing conventions for labels, statuses, alerts, and action copy.

## Voice

- Direct, operational, and neutral.
- Prefer specific nouns (Work Order, Discovery Case, Product Intent).
- Avoid marketing language and vague verbs.

## UI text rules

- Button labels start with explicit verbs: "Create", "Approve", "Assign", "Retry".
- Status labels are short and canonical (for example: `Blocked`, `In Progress`).
- Empty-state copy explains next best action.
- Error copy includes what failed and what user can do next.

## Terminology

Use canonical entity names from Foundry docs:

- Discovery Case
- Product Intent
- Work Order
- Governance Verdict
- Workspace Session

Do not introduce aliases in UI text without documenting the reason.

## Microcopy checklist

- Is this sentence action-oriented?
- Is the actor clear?
- Is domain vocabulary consistent with existing docs?
- Can this be scanned quickly in a high-density console?
