## Document Structure

Each persona README follows this structure:

1. **Primary Workspace** — The Workspace this persona primarily operates in
2. **Role** — Brief description of what this persona does
3. **Primary Jobs** — Core JTBD with acceptance criteria
4. **Supporting Jobs** — Secondary JTBD (less detailed)
5. **Open Questions** — Areas needing further specification
