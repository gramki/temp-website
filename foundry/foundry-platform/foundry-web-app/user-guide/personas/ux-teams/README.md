# UX Teams — Jobs to be Done

**Primary Workspace:** UX Design

**Role:** Design the user experience — create wireframes, prototypes, design specs, and design system components.

## Prerequisites

- Access to a Workbench with UX Design Workspace membership
- Figma integration configured (if using Figma for design assets)
- Understanding of [Work Order lifecycle](../../../../work-order-runtime/user-guide/work-order-lifecycle.md)

---

## Primary Jobs

### J1. View design-related Work Orders and Human Tasks
**When** I start my work session  
**I want to** see all UX Design Work Orders and Human Tasks assigned to me  
**So that** I can prioritize design work

**Acceptance Criteria:**
- List of UX Work Orders with status, priority
- Filter by Workbench, Track, status
- Human Tasks queue
- Visual indicator of design stage (wireframe, prototype, spec)

---

### J2. Create/refine design artifacts
**When** a Work Order requires design output  
**I want to** create or refine wireframes, prototypes, and design specs  
**So that** Development has what they need

**Acceptance Criteria:**
- Design artifact editor/uploader
- Version history
- Link to Product Intent and spec
- Preview capability
- Agent-assisted design suggestions

---

### J3. Review agent-generated design suggestions
**When** an agent produces design options  
**I want to** review, select, or modify the suggestions  
**So that** the design meets quality standards

**Acceptance Criteria:**
- Side-by-side comparison of options
- Select/reject/modify actions
- Feedback to agent for iteration
- Rationale capture

---

### J4. Collaborate on design iterations with Product/Development
**When** designs need feedback  
**I want to** share designs and gather feedback  
**So that** designs are validated before implementation

**Acceptance Criteria:**
- Share design for review
- Collect comments/annotations
- Track feedback status (addressed/pending)
- Version comparison

---

### J5. Validate UX against specifications
**When** ensuring design-spec alignment  
**I want to** compare designs to specifications  
**So that** nothing is missed

**Acceptance Criteria:**
- Spec checklist mapped to design elements
- Gap identification
- Compliance indicators

---

### J6. Approve design transitions to Development
**When** a design is ready for implementation  
**I want to** approve the transition  
**So that** Development can proceed

**Acceptance Criteria:**
- Design review checklist
- Approve/reject/request-changes actions
- Handoff notes
- Asset export/links

---

## Supporting Jobs

### J7. Access Design repository
**When** I need design assets, patterns, or guidelines  
**I want to** browse and search the Design repository  
**So that** I can find and reuse components

---

### J8. Track design feedback and iterations
**When** managing design revisions  
**I want to** see feedback history and iteration count  
**So that** I can track design evolution

---

### J9. View user research and feedback data
**When** informing design decisions  
**I want to** access user research and feedback  
**So that** designs are user-informed

---

### J10. Search Work Orders by design status
**When** looking for specific design work  
**I want to** filter Work Orders by design status  
**So that** I can find them quickly

---

### J11. Review UX-related defects from QA
**When** QA reports UX issues  
**I want to** see and address them  
**So that** designs are corrected

---

### J12. Manage design system components
**When** maintaining the design system  
**I want to** add, update, or deprecate components  
**So that** the system stays current
