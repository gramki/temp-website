**Track:** Run — deploying and operating artifacts in production.

| Workspace | Folder | Example Scenarios |
|-----------|--------|-------------------|
| Product Specification | [product-specification/](product-specification/) | Operational requirements update |
| UX Design | [ux-design/](ux-design/) | (typically not active in Run) |

| Release | [release/](release/) | Deployment execution, Rollback |
| Governance | [governance/](governance/) | Deployment approval, Incident escalation |

## Adding Scenarios

Each Workspace folder will contain scenario definitions as they are authored. A scenario definition includes:

- Scenario name and description
- Inputs and outputs
- Skills required
- Knowledge required
- Tools required
- Success criteria
- Agent recommendations
