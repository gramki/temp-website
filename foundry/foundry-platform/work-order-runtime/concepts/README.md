# Work Order Runtime Concepts

This folder contains module-specific concept definitions for the Work Order Runtime. These concepts describe WO Runtime internals — components, patterns, and mechanisms that belong to this module.

## How to use these concepts

- **Module-scoped.** These concepts explain WO Runtime implementation details.
- **Link to platform concepts.** When a concept references a platform-wide construct (Work Order, Task, Scenario, etc.), link to [../concepts/](../../concepts/).
- **Complement, don't duplicate.** If you need to explain what a Task *is*, link to the platform concept. If you need to explain how the Task Manager *schedules* Tasks, that's module-specific.

## Concept Index

| Concept | Definition |
|---------|------------|
| [Management Plane Interface](management-plane-interface.md) | Protocol for liveness ack, heartbeat, and shutdown with Session Management |
| [WO Runtime Daemon](wo-runtime-daemon.md) | Long-running process within each Workspace Session that coordinates WO execution |
| [Task Manager](task-manager.md) | Component that manages task trees, dependencies, and state transitions |
| [Agent Spawner](agent-spawner.md) | Component that prepares execution harnesses and spawns Employed Agents |
| [Context Compilation](context-compilation.md) | Process of assembling hierarchical knowledge into agent context |
| [Completion Reporter](completion-reporter.md) | Component that notifies the Orchestrator when Work Orders complete |
| [Local State Store](local-state-store.md) | Per-session SQLite database for WO and task state |

## Relationship to Platform Concepts

These module concepts build on platform-wide concepts defined in [../../concepts/](../../concepts/):

| Platform Concept | WO Runtime Realization |
|------------------|------------------------|
| [Work Order](../../concepts/work-order.md) | Attached and executed by WO Runtime Daemon |
| [Task](../../concepts/task.md) | Managed by Task Manager; executed by agents or humans |
| [Scenario](../../concepts/scenario.md) | Read by Agent Spawner to determine Skilled Agent and execution |
| [Agent Model](../../concepts/agent-model.md) | Employed Agents spawned by Agent Spawner |
| [Workspace Session](../../concepts/workspace-session.md) | In-session worker; liveness ack to Session Management |
| [Delegation](../../concepts/delegation.md) | Tokens generated by Agent Spawner for Employed Agents |
