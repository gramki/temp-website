# Definitions, Metrics, and Canonical KPIs

## What this is

This knowledge type captures the enterprise’s **asserted measurement meaning**:

- definitions (“what does MAU mean?”)
- metric formulas
- canonical KPIs and dimensions

It answers:

> *“When we measure or report, what do these numbers *mean*?”*

## Typical artifacts

- KPI definitions (with scope, exclusions, formula, owner)
- canonical dashboards and scorecards (when treated as knowledge-of-record)
- data dictionaries and semantic layer definitions

## Management characteristics

- **Owned semantics**: a metric is not canonical until ownership and meaning are explicit.
- **Versioned**: metric definitions change; old definitions must remain accessible for historical comparisons and audits.
- **Tested**: metric computations should be validated like software (unit tests + data quality checks).
